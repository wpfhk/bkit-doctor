# bkit-doctor Automation Skill

> Auto-generated by `bkit-doctor skill`. This file teaches Claude Code how to use bkit-doctor automatically.

## Rules

### RULE 1: PROACTIVE DOCUMENTATION
Do not wait for the user to ask for documentation. When a new task, feature, or bugfix is given, automatically run:
```bash
npx bkit-doctor pdca-plan "<topic>" --type <feature|bugfix|refactor|guideline>
```
Fill in the generated plan file before writing any code.

### RULE 2: STATE SYNC
Before writing any code, always check the current documentation state:
```bash
npx bkit-doctor pdca-list
```
If a PDCA plan already exists for the current topic, read it and follow the plan. If `.bkit-doctor/pdca-state.json` exists, respect the current stage.

### RULE 3: PIPELINE
When you finish coding a task, automatically run the remaining PDCA stages without asking the user:
```bash
npx bkit-doctor pdca-do "<topic>"
npx bkit-doctor pdca-check "<topic>"
```
If all checks pass, run:
```bash
npx bkit-doctor pdca-report "<topic>"
```

## Available Commands

| Command | Purpose |
|---------|---------|
| `bkit-doctor check` | Diagnose project structure |
| `bkit-doctor fix --yes` | Auto-fix structural issues |
| `bkit-doctor pdca "<topic>"` | Generate full PDCA guide |
| `bkit-doctor pdca-plan "<topic>"` | Plan document only |
| `bkit-doctor pdca-do "<topic>"` | Do document only |
| `bkit-doctor pdca-check "<topic>"` | Check document only |
| `bkit-doctor pdca-report "<topic>"` | Report document only |
| `bkit-doctor pdca-list` | List all PDCA documents |
| `bkit-doctor setup` | Interactive project setup wizard |

## Workflow

```
User gives task → RULE 1 (plan) → Implement → RULE 3 (do + check + report)
                   ↑                              ↓
              RULE 2 (state sync)          Auto-documented ✓
```
